LCS最长公共子串

LCS（最长连续公共子串）算法

假设我们有两个字符串 ABCDMARK1234 ， MARKDOWN34

那么最长公共子串就是MARK

我们建立假设原来的子串分别为A，B，我们建立一个新的数组C，去记录最长的连续子串的位置。

递推公式：
C[i,j] = 0 (A[i]不等于B[j])
= 1 (A[i]等于B[i]且i或j小于1)
= C[i-1,j-1] + 1 (A[i]等于B[j])

使用m，n记录C数组中最大的位置

public class Main {

public static String A = "ABCDMARK1234" ;
public static String B = "MARKDOWN" ;
public static int[][] C ;
public static int m;
public static int n;
public static int maxValue = 0;
public static void main(String[] args)
{
    /* 创建二维数组C */
    C = new int[A.length()][B.length()];
    /* i,j循环 */
    for (int i = 0 ; i<A.length();i++)
        for (int j =0 ; j<B.length(); j++)
        {
            if (A.charAt(i) == B.charAt(j)) {
                /* 如果i或者j小于1，那么当前的值直接置为1 */
                if (i<1 || j <1)
                    C[i][j] = 1;
                else{
                    C[i][j] = C[i - 1][j - 1] + 1;
                }

                if (C[i][j] > maxValue)
                {
                    maxValue = C[i][j];
                    m = i;
                    n = j;
                }
            }
            else
                C[i][j] = 0;

        }
    /* 将m和n指向匹配第一个字符的位置 */
    m = m - maxValue + 1 ;
    n = n - maxValue + 1 ;
    /* 循环输出 */
    for (int i=0;i<maxValue;i++)
    {
        System.out.print(A.charAt(m));
        m++;
    }
}
}

算法-矩阵链乘积

矩阵链问题描述：

给定n个矩阵构成的一个链，其中i=1,2,…n，矩阵A的维数为pi-1 pi ，对乘积 A1 A2 …An 以一种最小化标量乘法次数的方式进行加全部括号。

括号其乘积的顺序是会影响到需计算乘积所需简单算术运算的数目，即其效率。例如，设A为一10×30矩阵，B为30×5矩阵与C为5×60矩阵，则
(AB)C 有 (10×30×5) + (10×5×60) = 1500 + 3000 = 4500 个运算
A(BC) 有 (30×5×60) + (10×30×60) = 9000 + 18000 = 27000 个运算

我们的目标就是找出来矩阵相乘的最佳顺序,得到最小的相乘次数

对于矩阵链问题，最好的解决方法是动态规划。

首先我们来找一下最优子结构：

我们需要计算整个矩阵链的的最优解，计算过程中，肯定有一个k（1 这两部分，寻找这两部分的最优解。

再具体一点，最优子结构就是指：矩阵链的最优解如果是在k(i和也是最优解。我们可以根据把问题分割，以构建总问题的最优解。

我们再来寻找状态和状态转移方程：

我们确定一个状态m[i,j] ,这指的是计算矩阵链 所需要的乘法次数的最小数目。

我们在按照之前的思路进行分解，以k做左右两边的分解，我们就可以得到递归式：

m[i,j] = 0 (i=j)
= min(i<=k<j) {m[i,k]+m[k+1,j]+pi-1pkpj} (i<j)

我们把k从i到j都试一次，取最小值。

这个时候，我们如果使用递归求解，时间复杂度会变为指数级，我们为了降低时间复杂度，可以再来开一个数组s[i][j]记录矩阵链取得最优时k的值。

代码如下，关于代码说明写在代码注释中：

/**
 * Created by libin on 14-5-15.
 */
public class Main {
/* 这里虽然是5个数字，但是实际上是4个矩阵 */
public static int Data[] = {3,5,2,1,10};
public static void main(String[] args)
{
  System.out.print("Result :" +  maxChain(Data));
}
public static int maxChain(int[] A)
{
    /* m 对应保存从Ai到Aj的最小的乘的次数 也就是从第i个矩阵到第j个矩阵，所以A[0]的元素都是没用的 */
    int[][] m = new int[A.length][A.length];
    /* s 对应保存从Ai到Aj的最佳的k ，和m一样，数组第一行是无效的*/
    int[][] s = new int[A.length][A.length];
    /* 设定一个大数，为了之前把数组m里面所有的值标记为无限大 */
    int MAX = 10000000;
    /* n是数组最大下标，这个值可以省略掉 */
    int n = A.length-1;
    /* 我们事前首先把所有的数字置成0 */
    for(int i=1;i<n+1;i++){
        m[i][i]=0;
    }
    /* l是矩阵的个数，只有两个以上矩阵相乘的时候才会考虑矩阵链乘积的问题，所以直接将l置为2*/
    for (int l = 2; l < A.length;l++)
    {
        /*i从1开始进行，因为本次循环会有l个矩阵参与运算，所以到length - l + 1 的时候循环终止*/
        for (int i = 1;i<A.length-l+1;i++)
        {
            /* 我们把数组m里面每个有效的值置为无限大 j是终结的位置，m[i][j]记录的是从i开始，到j结束的算术乘动作最小值*/
            int j = i + l - 1;
            m[i][j] = MAX;
            /* 我们来遍历所有可能的k，找一下最合适的分割点 */
            for (int k = i ; k< j ;k++)
            {
                int result = m[i][k] + m[k+1][j] + A[i-1]*A[k]*A[j];
                if (result < m[i][j])
                {
                    m[i][j] = result;
                    s[i][j] = k;
                }

            }

        }
    }
    System.out.print("display m :" + "\n");
    display(m);
    System.out.print("---------------" + "\n");
    System.out.print("display s:" + "\n");
    display(s);
    /*因为第一行无效，所以我们实际上返回的是第二行数组中最后一个值 对于样例数据来说，i=1,j=4 */
    return m[1][A.length-1];
}

/**
 * 测试所用，适用于打印m或者s数组
 * @param array
 */
static void display(int [][] array){
    int n = array[0].length;
    for(int i=0;i<n;i++){
        for(int j=0;j<n;j++){

                System.out.print(array[i][j]+" ");
        }
        System.out.print("\n");
    }
}
}

算法-动态规划相关

动态规划是一种很长见的算法，经典的题目如LCS，LIS都会用到动态规划。

动态规划的特点是，动态规划会存储上一个状态的值，用递推公式去推下一个状态，保存起来，最终来得到最优解。

我们需要找到某个状态的最优解，推下去得到其他状态的最优解。

我们来通过两个经典的背包问题来说明动态规划：

0-1背包问题

There are 5 items that have a value and weight list below, the knapsack can contain at most 100 Lbs

values: 20 30 65 40 60

weight: 10 20 30 40 50

Solve the problem as 0/1 knapsack.

0-1背包问题，就是有一个容量是V的背包，第i件的重量是w[i],价值是v[i],每个物品只能装一次，那么如何让背包能装的价值最大

理论上这个问题也可以用贪心去解决，但是贪心不一定能得出来合适的解，我们使用动态规划来解决这个问题

0-1背包，指的就是对于一个物品只有两个选择：放，或者不放

动态规划需要明确两个方面：状态和状态转移方程

我们使用f[i][v]表示前i个物品放入容量为v的背包可以获得的最大价值
f[i][v] 就是一个状态

状态转移方程是：

f[i][v] = max{f[i-1][v]，f[i-1][v-w[i]]+v[i]}

f[i-1][v] 就是这个物品不放的情况下状态

f[i-1][v-w[i]] + w[i] 就是在放这个物品的情况下的状态

贴上代码，详情在注释中有体现：

public class Main {
/* 容量 */
public static int V = 100;
/* 物品总数 */
public static int I = 5;
/* 各个物品的values */
int[] values = {20,30,65,40,60};
/* 各个物品的weight */
int[] weight = {10,20,30,40,50};
/* 建立储存状态的二维数组 ，表示在V容量装前I个物品的价值，加了1是为了让数组下标I就是实际的件数，V就是实际的容量*/
int[][] f = new int[I+1][V+1];

public int knapSack()
{
    /* i代表装物品的件数，我们可以直接从1开始循环，因为i=0的时候没有意义（一个物品都不装，肯定价值是0）*/
    for (int i=1;i<=I;i++)
    {
        /* j代表容量，容量是0的时候也没有意义，所以直接从1开始循环 */
        for (int j=1;j<=V;j++) {
                /*如果我没有装的下这个物品的容量，那么和之前的价值是一样的*/
                if (j < weight[i-1])
                    f[i][j] = f[i][j - 1];
                else
                    /*我有了装的下这个物品的容量，那么我们就上我们的递归公式，在装和不装中取一个和谐的值 */
                    f[i][j] = max(f[i - 1][j], f[i - 1][j - weight[i-1]] + values[i-1]);
                /*测试输出每次的i,j,f */
                System.out.print("i:" + i + " j:" + j + " f:" + f[i][j] + "\n");

        }
    }
    /* 返回最大值 */
    return f[I][V];
}
/* 取两个数大的那一个 */
public int max(int a,int b)
{
    if (a< b )
        return b;
    else
        return a;
}

public static void main(String[] args)
{
    Main main = new Main();
    System.out.print(main.knapSack());

}
}

分治法的一道题

题目：

Give a divide and conquer algorithm for the following problem: you are given two sorted lists of size m and n, and are allowed unit time access to the ith element of each list. Give an O(lg m + lgn) time algorithm for computing the kth largest element in the union of the two lists. (For simplicity, you can assume that the elements of the two lists are distinct).

翻译过来，就是说，使用分治法解决一个问题：给两个已经排序好的数组，如何找出对于这两个数组合并、排序之后来说第k大的元素，要求时间复杂度为O(logm+logn)

这个问题最直接的方法就是用归并排序，生成一个新的数组（毕竟数据源是两个有序数组），然后对新的数组遍历到k，返回这个数，结束，这种方法简单粗暴，但是我们有更好的方法。

我们可以不去选择做“排序”这个动作，排序消耗大量时间，而归并排序又需要消耗大量的空间。我们来用另外一种方式来考虑这个问题。

首先，我们对比一下A[k/2-1]和B[k/2-1]的值：

  • 如果A[k/2-1]<B[k/2-1],那么A[0]-A[k/2-1]这些值都可以被忽略掉了

    假设不能忽略，即A∪B[K-1]就在A[0…K/2-1]里面，取最大值A[K/2-1]为A∪B[K-1]。
    由此则B中必定有K/2个元素小于A[K/2-1]，而又因为A[K/2-1] < B[K/2-1]，故假设不成立。

  • 如果A[k/2-1]>B[k/2-1],那么同理，B[0]-B[k/2-1]这些值夜都可以被忽略掉了
  • 如果A[k/2-1] = B[k/2-1],那么我们就找到了我们想要的那个值了

如果A[k/2-1]和B[k/2-1]不相等，我们就可以去掉K/2个元素

接下来，A或者B变为去掉K/2的元素的新数组，比如如果A[k/2-1]<B[k/2-1]，那么 A= A[k/2] - A[A.length]

然后，K = K - K/2 ,因为我们已经去掉了k/2个元素（这样写是因为比K = K/2更好理解一些）

一直递归下去，执行A和B数组中某个元素的比较，直到A或者B数组长度为0或者1。

如果为0，那么就取另外一个数组的下标为k/2的元素即可，如果是1，那么就取A[0]和B[0]中比较小的元素。

实际代码：

 int find(int[] A, int[] B, int k) {
    //避免A或者B是个空的
    if (A.length == 0) return B[k - 1];
    if (B.length == 0) return A[k - 1];
    //递归到k =1的情况
    if (k == 1) return Math.min(A[0], B[0]);
    //如果A比B还长的话...
    if (A.length > B.length) return find(B, A, k);
    //如果k/2比a的长度还要长，那么就取A的长度
    int a = Math.min(A.length,k/2);
    int b = k - a;
    //我们的主要步骤
    if (A[a-1] < B[b-1]) {
        return find(Arrays.copyOfRange(A, a, A.length), B, k - a);
    } else if (A[a-1] > B[b-1]) {
        return find(A, Arrays.copyOfRange(B, b, B.length), k - b);
    } else {
        return A[a-1];
    }
}

我们来看看分治法具体的三个步骤的体现:

  • 分 —— A或者B的下标为K/2-1前面的部分和后面的部分分别处理，递归到A或者B数组里面只剩下0或者1个元素
  • 治 —— 比较A[k/2-1]和B[k/2-1]的值，大于、小于、等于分别有三种的处理方法
  • 合 —— 感觉这里体现的不是很明显，最后返回的是一个值，没有具体的合并过程

算法-分治法在归并排序和快速排序中的体现

分治法，就是把一个大的问题分解成若干个子问题，子问题可以进一步分解为更小的子问题，直到问题分解为一个可以直接求解的子问题，然后把子问题的解合并即可得到大问题的解，分而治之，各个击破。基本上分治法都得基于递归去解决。

分治法保持分、治、合三个步骤求解。

分治法在归并排序和快速排序得到具体的体现，我们来研究一下

归并排序

以下代码引自维基百科-归并排序

public static int[] mergeSort(int[] arr){//归并排序 --递归
if(arr.length==1){
    return arr;
}
int half = arr.length/2;
int[] arr1 = new int[half];
int[] arr2 = new int[arr.length-half];
System.arraycopy(arr, 0, arr1, 0, arr1.length);
System.arraycopy(arr, half, arr2, 0, arr2.length);
arr1 = mergeSort(arr1);
arr2 = mergeSort(arr2);
return mergeSortSub(arr1,arr2);
}

private static int[] mergeSortSub(int[] arr1,int[] arr2){//归并排序子程序
int[] result = new int[arr1.length+arr2.length];
int i = 0;
int j = 0;
int k = 0;
while(true){
    if(arr1[i] < arr2[j]){
        result[k] = arr1[i];
        if(++i>arr1.length-1){
            break;
        }
    }else{
        result[k] = arr2[j];
        if(++j>arr2.length-1){
            break;
        }
    }
    k++;
}
for(;i<arr1.length;i++){
    result[++k] = arr1[i];
}
for(;j<arr2.length;j++){
    result[++k] = arr2[j];
}
return result;
}

我们来具体看看是如何“分治“是如何在归并排序里面体现的

  1. 首先我们要理解归并排序的做法：简单的说，就是把两个已经排序完毕的数组合并成一个有序数组。 形象一点说，就是把两摞已经排好顺序的扑克牌合并成一摞。 假设这两摞牌都是升序排列（也就是说最上面的牌是最小的），那么，做这个排序的事情的时候，我们比较两摞牌最上面的牌的大小，把比较小的放在另外一摞的第一个，然后继续做比较，把小的放在新的牌堆底，直到某一摞牌都被放完了，空了，我们把另外一摞所有的牌都放在新的牌堆的底部。
  2. 在算法上，首先我们把一个数组平分为两部分，我们称之为arr1和arr2，我们开始进行递归，将arr1和arr2无限的分解，直到他们只被分解的剩下了1个元素，而每次递归的子程序都会执行归并排序
  3. 我们设定一个归并排序的子程序，传入两个数组，进行归并排序。我们开辟一个数组int[] result = new int[arr1.length+arr2.length];，长度是传入的两个数组长度之和。我们开启三个变量，i，j，k。 i的意义是传入第一个数组遍历到的位置，j为传入的第二个数组遍历到的位置，k是我们开辟的新数组指向的位置。接下来的操作就跟我们开始的时候说的“扑克牌”的操作类似，把传入两个数组大的那个数放到我们新开辟的数组里面，直到某个数组里面没有元素了（i或者j等于所指向的数组的长度），跳出循环，将没遍历完的数组的所有元素加入到新开辟的数组中，把新开辟的数组返回，归并排序子程序结束。
  4. 到现在，整个流程就比较清晰了，分解为直到数组里面剩了1个元素，从这里开始进行排序（1个数字肯定是有序的），接下来是两个数字的归并排序，3个数字的归并排序，以此类推，最终执行完整个数组的归并排序。

我们来看一下分、治、和在归并排序里面的具体体现：

  • 分 —— 分在归并排序里面就是无限把一个数组分成两个数组，直到分成的新数组里面剩了1个元素
  • 治 —— 主要是指归并排序子程序中对比两个数组i/j所指的大小，并加入到新数组中
  • 合 —— 在归并排序子程序中两个有序数组合并成一个有序数组，并返回

快速排序

相同的道理，快速排序也是类似的实现方式

public static void quicksort(int n[], int left, int right) {
    int dp;
    if (left < right) {
        dp = partition(n, left, right);
        quicksort(n, left, dp - 1);
        quicksort(n, dp + 1, right);
    }
}

public static int partition(int n[], int left, int right) {
    int pivot = n[left];
    while (left < right) {
        while (left < right && n[right] >= pivot)
            right--;
        if (left < right)
            n[left++] = n[right];
        while (left < right && n[left] <= pivot)
            left++;
        if (left < right)
            n[right--] = n[left];
    }
    n[left] = pivot;
    return left;
}

快速排序的步骤可以这样子理解：

  1. 在数组里面挑选出来一个元素，作为基准（pivot），这里挑这个元素，这里的可以选择挑选第一个。不过，最好是随机的选一个，做一个随机化快速排序，因为挑选的基准会影响到排序的效率，如果说挑选的是一个元素恰好是有序数组的第一个或者最后一个，会造成快速排序的最坏情况（O(n^2)）。
  2. 把数组里面比基准值大的元素放在后边，比基准值小的放在前面，相同的放在任意一边。
  3. 把基准值左边和右边的数组再当成一个新的数组，从第一步开始执行，直到递归到最后数组的大小是0或者1。

最后，我们来看一下在快速排序里面分治法的体现

  • 分 —— 基准元素左右两边各作为一个子数组
  • 治 —— 体现在partition中，分区，返回一个基准值所对应的位置
  • 合 —— 所有的操作实际上都是在原始数组中做的操作，直接会返回这个排序好的数组
