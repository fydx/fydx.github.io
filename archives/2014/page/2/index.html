java接口相关

接口是Java里面很常用的一个特性，接口的作用是用来定义一个规范，让其他的类实现这个接口，让我们明确，这个类实现了这个接口，可以做这个事情。

举个例子，我们中午在街上，肚子饿了，附近有很多写着“餐厅”招牌的地方。那么这些地方实际上就是实现了“有吃的东西卖”这个接口，告诉我们这个地方有饭吃。至于里面的饭是什么，好不好吃，那么就是看各家的实现了。

我们来按这个例子写一个代码

我们定义了一个接口SellFood，创建两个类DumplingRestaurant和NoodleResuaurant去实现这个接口，创建一个Person可以去买食物

public class interfaceTest {


public interface SellFood
{
    public void sell();

}
public class DumplingRestaurant implements SellFood
{
    @Override
    public void sell()
    {
        System.out.print("Sell Dumplings Here" + "\n");
    }

}
public class NoodleRestaurant implements SellFood
{
    @Override
    public void sell()
    {
        System.out.print("Sell Noodle Here" + "\n");
    }
}
public class Person
{
    public void buy(SellFood sellFood)
    {
        sellFood.sell();
    }
}
public static void main(String[] args)
{
    interfaceTest interfaceTest =  new interfaceTest();
    DumplingRestaurant dumplingRestaurant = interfaceTest.new DumplingRestaurant();
    NoodleRestaurant noodleRestaurant = interfaceTest.new NoodleRestaurant();
    Person person = interfaceTest.new Person();
    person.buy(dumplingRestaurant);
    person.buy(noodleRestaurant);
}

}

从这个代码我们可以看出来，Person是去“可以买的东西地方”买东西，没必要去关心这里到底是卖什么东西，只要这个地方实现了SellFood的接口就可以被传入到buy这个方法里面

接口最典型的应用就是Comparable和Comparator了

对于Comparable来说：

实现Comparable，就需要类里面实现compareTo方法，明确这个类是可以被排序的，当这个类被加载到某个集合（比如数组或者List中）的时候，可以使用Collections.sort()和Arrays.sort()排序

compareTo方法可以这样子调用

    @Override
    public int compareTo(Person person) {
        return this.age - person.age;
    }

返回 >0 =0 <0 的数以说明该对象相对于另外一个对象大于，等于，或者小于

对于Comparator来说：

Comparable是在类内部来实现的，而Comparator是在集合的外部实现的，Comparator不需要在类的内部去定义，在类的外部创建即可

public static class AgeComparator implements Comparator<Person> {

@Override
public int compare(Person p1, Person p2) {
    return p1.getAge() - p2.getAge();
}
}

使用Comparator的时候需要这样写：

 ArrayList<Person> persons = new ArrayList<Person>();
 person.add("12");
 // 增加几个元素之后
 Collections.sort(persons, comparator);

这个意思是，必须先进行外部类的实例化，才能通过外部类来引用内部类。因为两个Restautant和Person都是内部类，所以我们必须先进行外部类的实例化，用这种方式来创建对象才行：

    DumplingRestaurant dumplingRestaurant = interfaceTest.new DumplingRestaurant();

分治法的一道题

题目：

Give a divide and conquer algorithm for the following problem: you are given two sorted lists of size m and n, and are allowed unit time access to the ith element of each list. Give an O(lg m + lgn) time algorithm for computing the kth largest element in the union of the two lists. (For simplicity, you can assume that the elements of the two lists are distinct).

翻译过来，就是说，使用分治法解决一个问题：给两个已经排序好的数组，如何找出对于这两个数组合并、排序之后来说第k大的元素，要求时间复杂度为O(logm+logn)

这个问题最直接的方法就是用归并排序，生成一个新的数组（毕竟数据源是两个有序数组），然后对新的数组遍历到k，返回这个数，结束，这种方法简单粗暴，但是我们有更好的方法。

我们可以不去选择做“排序”这个动作，排序消耗大量时间，而归并排序又需要消耗大量的空间。我们来用另外一种方式来考虑这个问题。

首先，我们对比一下A[k/2-1]和B[k/2-1]的值：

  • 如果A[k/2-1]<B[k/2-1],那么A[0]-A[k/2-1]这些值都可以被忽略掉了

    假设不能忽略，即A∪B[K-1]就在A[0…K/2-1]里面，取最大值A[K/2-1]为A∪B[K-1]。
    由此则B中必定有K/2个元素小于A[K/2-1]，而又因为A[K/2-1] < B[K/2-1]，故假设不成立。

  • 如果A[k/2-1]>B[k/2-1],那么同理，B[0]-B[k/2-1]这些值夜都可以被忽略掉了
  • 如果A[k/2-1] = B[k/2-1],那么我们就找到了我们想要的那个值了

如果A[k/2-1]和B[k/2-1]不相等，我们就可以去掉K/2个元素

接下来，A或者B变为去掉K/2的元素的新数组，比如如果A[k/2-1]<B[k/2-1]，那么 A= A[k/2] - A[A.length]

然后，K = K - K/2 ,因为我们已经去掉了k/2个元素（这样写是因为比K = K/2更好理解一些）

一直递归下去，执行A和B数组中某个元素的比较，直到A或者B数组长度为0或者1。

如果为0，那么就取另外一个数组的下标为k/2的元素即可，如果是1，那么就取A[0]和B[0]中比较小的元素。

实际代码：

 int find(int[] A, int[] B, int k) {
    //避免A或者B是个空的
    if (A.length == 0) return B[k - 1];
    if (B.length == 0) return A[k - 1];
    //递归到k =1的情况
    if (k == 1) return Math.min(A[0], B[0]);
    //如果A比B还长的话...
    if (A.length > B.length) return find(B, A, k);
    //如果k/2比a的长度还要长，那么就取A的长度
    int a = Math.min(A.length,k/2);
    int b = k - a;
    //我们的主要步骤
    if (A[a-1] < B[b-1]) {
        return find(Arrays.copyOfRange(A, a, A.length), B, k - a);
    } else if (A[a-1] > B[b-1]) {
        return find(A, Arrays.copyOfRange(B, b, B.length), k - b);
    } else {
        return A[a-1];
    }
}

我们来看看分治法具体的三个步骤的体现:

  • 分 —— A或者B的下标为K/2-1前面的部分和后面的部分分别处理，递归到A或者B数组里面只剩下0或者1个元素
  • 治 —— 比较A[k/2-1]和B[k/2-1]的值，大于、小于、等于分别有三种的处理方法
  • 合 —— 感觉这里体现的不是很明显，最后返回的是一个值，没有具体的合并过程

算法-分治法在归并排序和快速排序中的体现

分治法，就是把一个大的问题分解成若干个子问题，子问题可以进一步分解为更小的子问题，直到问题分解为一个可以直接求解的子问题，然后把子问题的解合并即可得到大问题的解，分而治之，各个击破。基本上分治法都得基于递归去解决。

分治法保持分、治、合三个步骤求解。

分治法在归并排序和快速排序得到具体的体现，我们来研究一下

归并排序

以下代码引自维基百科-归并排序

public static int[] mergeSort(int[] arr){//归并排序 --递归
if(arr.length==1){
    return arr;
}
int half = arr.length/2;
int[] arr1 = new int[half];
int[] arr2 = new int[arr.length-half];
System.arraycopy(arr, 0, arr1, 0, arr1.length);
System.arraycopy(arr, half, arr2, 0, arr2.length);
arr1 = mergeSort(arr1);
arr2 = mergeSort(arr2);
return mergeSortSub(arr1,arr2);
}

private static int[] mergeSortSub(int[] arr1,int[] arr2){//归并排序子程序
int[] result = new int[arr1.length+arr2.length];
int i = 0;
int j = 0;
int k = 0;
while(true){
    if(arr1[i] < arr2[j]){
        result[k] = arr1[i];
        if(++i>arr1.length-1){
            break;
        }
    }else{
        result[k] = arr2[j];
        if(++j>arr2.length-1){
            break;
        }
    }
    k++;
}
for(;i<arr1.length;i++){
    result[++k] = arr1[i];
}
for(;j<arr2.length;j++){
    result[++k] = arr2[j];
}
return result;
}

我们来具体看看是如何“分治“是如何在归并排序里面体现的

  1. 首先我们要理解归并排序的做法：简单的说，就是把两个已经排序完毕的数组合并成一个有序数组。 形象一点说，就是把两摞已经排好顺序的扑克牌合并成一摞。 假设这两摞牌都是升序排列（也就是说最上面的牌是最小的），那么，做这个排序的事情的时候，我们比较两摞牌最上面的牌的大小，把比较小的放在另外一摞的第一个，然后继续做比较，把小的放在新的牌堆底，直到某一摞牌都被放完了，空了，我们把另外一摞所有的牌都放在新的牌堆的底部。
  2. 在算法上，首先我们把一个数组平分为两部分，我们称之为arr1和arr2，我们开始进行递归，将arr1和arr2无限的分解，直到他们只被分解的剩下了1个元素，而每次递归的子程序都会执行归并排序
  3. 我们设定一个归并排序的子程序，传入两个数组，进行归并排序。我们开辟一个数组int[] result = new int[arr1.length+arr2.length];，长度是传入的两个数组长度之和。我们开启三个变量，i，j，k。 i的意义是传入第一个数组遍历到的位置，j为传入的第二个数组遍历到的位置，k是我们开辟的新数组指向的位置。接下来的操作就跟我们开始的时候说的“扑克牌”的操作类似，把传入两个数组大的那个数放到我们新开辟的数组里面，直到某个数组里面没有元素了（i或者j等于所指向的数组的长度），跳出循环，将没遍历完的数组的所有元素加入到新开辟的数组中，把新开辟的数组返回，归并排序子程序结束。
  4. 到现在，整个流程就比较清晰了，分解为直到数组里面剩了1个元素，从这里开始进行排序（1个数字肯定是有序的），接下来是两个数字的归并排序，3个数字的归并排序，以此类推，最终执行完整个数组的归并排序。

我们来看一下分、治、和在归并排序里面的具体体现：

  • 分 —— 分在归并排序里面就是无限把一个数组分成两个数组，直到分成的新数组里面剩了1个元素
  • 治 —— 主要是指归并排序子程序中对比两个数组i/j所指的大小，并加入到新数组中
  • 合 —— 在归并排序子程序中两个有序数组合并成一个有序数组，并返回

快速排序

相同的道理，快速排序也是类似的实现方式

public static void quicksort(int n[], int left, int right) {
    int dp;
    if (left < right) {
        dp = partition(n, left, right);
        quicksort(n, left, dp - 1);
        quicksort(n, dp + 1, right);
    }
}

public static int partition(int n[], int left, int right) {
    int pivot = n[left];
    while (left < right) {
        while (left < right && n[right] >= pivot)
            right--;
        if (left < right)
            n[left++] = n[right];
        while (left < right && n[left] <= pivot)
            left++;
        if (left < right)
            n[right--] = n[left];
    }
    n[left] = pivot;
    return left;
}

快速排序的步骤可以这样子理解：

  1. 在数组里面挑选出来一个元素，作为基准（pivot），这里挑这个元素，这里的可以选择挑选第一个。不过，最好是随机的选一个，做一个随机化快速排序，因为挑选的基准会影响到排序的效率，如果说挑选的是一个元素恰好是有序数组的第一个或者最后一个，会造成快速排序的最坏情况（O(n^2)）。
  2. 把数组里面比基准值大的元素放在后边，比基准值小的放在前面，相同的放在任意一边。
  3. 把基准值左边和右边的数组再当成一个新的数组，从第一步开始执行，直到递归到最后数组的大小是0或者1。

最后，我们来看一下在快速排序里面分治法的体现

  • 分 —— 基准元素左右两边各作为一个子数组
  • 治 —— 体现在partition中，分区，返回一个基准值所对应的位置
  • 合 —— 所有的操作实际上都是在原始数组中做的操作，直接会返回这个排序好的数组

Socket相关

Socket是网络交互主要使用的一种方式，我们用一个例子来说明。

这是一个QQ客户端，分为服务器端和客户端

服务器端代码如下

package chat;  

import java.io.BufferedReader;  
import java.io.IOException;  
import java.io.InputStreamReader;  
import java.io.PrintWriter;  
import java.net.ServerSocket;  
import java.net.Socket;  
import java.util.ArrayList;  
import java.util.List;  
import java.util.concurrent.ExecutorService;  
import java.util.concurrent.Executors;  

public class chat {  
    private static final int PORT = 8888; // 端口号  
    private static List<Socket> list = new ArrayList<Socket>(); // 保存连接对象
    private ExecutorService exec;
    private ServerSocket server;
public static void main(String[] args) {  
    new chat();  
}  

public chat() {  
    try {  
        server = new ServerSocket(PORT);  
        exec = Executors.newCachedThreadPool();  
        System.out.println("服务器已启动！");  

        Socket client = null;  
        while (true) {  
            client = server.accept(); // 接收客户连接  
            list.add(client);  
            exec.execute(new ChatTask(client));  
        }  
    } catch (IOException e) {  
        e.printStackTrace();  
    }  
}  

static class ChatTask implements Runnable {  
    private Socket socket;  
    private BufferedReader br;  
    private PrintWriter pw;  
    private String msg;  

    public ChatTask(Socket socket) throws IOException {  
        this.socket = socket;  
        br = new BufferedReader(new InputStreamReader(socket  
                .getInputStream()));  
        msg = "【" + this.socket.getInetAddress() + "】进入聊天室！当前聊天室有【"  
                + list.size() + "】人";  

        sendMessage();  
    }  

    public void run() {  
        try {  
            while ((msg = br.readLine()) != null) {  
                if (msg.trim().equals("bye")) {  
                    list.remove(socket);  
                    br.close();  
                    pw.close();  
                    msg = "【" + socket.getInetAddress() + "】离开聊天室！当前聊天室有【"  
                            + list.size() + "】人";  
                    socket.close();  

                    sendMessage();  
                    break;  
                } else {  
                    msg = "【" + socket.getInetAddress() + "】说：" + msg;  
                    sendMessage();  
                }  
            }  
        } catch (IOException e) {  
            e.printStackTrace();  
        }  
    }  

    /** 
     * 群发消息给聊天室的所有人 
     */  
    private void sendMessage() throws IOException {  
        System.out.println(msg);  
        for (Socket client : list) {  
            pw = new PrintWriter(client.getOutputStream(), true);  
            pw.println(msg);  
        }  
    }  
}  
}

/** 
 * 客户端线程获取控制台输入消息 
 */  
static class Sender implements Runnable {  
    private Socket socket;  

    public Sender(Socket socket) {  
        this.socket = socket;  
    }  

    public void run() {  
        try {  
            BufferedReader br = new BufferedReader(new InputStreamReader(

            e.printStackTrace();  
        }

Class.forName("com.mysql.jdbc.Driver");// 加载Mysql数据驱动

首先，我们需要加载mysql的驱动
这里的Class.forName 将class进行解释和初始化，需要在DriveManager里面注册，以初始化驱动

2，创建数据库连接

 Connection connection;
 connection = DriverManager.getConnection(
                "jdbc:mysql://localhost:3306/testjdbc", "root", "");// 创建数据连接，参数为：连接的ip和端口+数据库的名字，mysql用户名，mysql用户名的密码

这里我们需要连接到这个数据库，getconnection的三个参数，一个是uri，使用mysql的话，就是“jdbc:mysql//“ + ip +”:” + 端口 + “/” + 数据库名，第二个参数是用户名，第三个参数是对应用户名的密码

3, 创建Statement对象 

  Statement statement ;
    try {
       statement = connection.createStatement();
                } catch (SQLException e) {

        e.printStackTrace();

    }

Statement对象是我们的操纵杆，我们操作数据库需要通过Statement对象来完成

4，执行SQL语句

我们创建了Statement的对象之后就可以执行SQL语句了，执行查询的方式就是statement.executeQuery(String sql) 这种形式，比如：

 try {
       statement = connection.createStatement();
        ResultSet resultSet =statement.executeQuery("select * from test");
        while (resultSet.next())
        {
            String name = resultSet.getString("name");
            int id = resultSet.getInt("id");
            System.out.print(name + "  " + id +"\n") ;
        }
    } catch (SQLException e) {

        e.printStackTrace();

    }

这里需要注意的一点是，如果执行的sql语句有返回值，那么返回的是一个ResultSet对象，使用方式和迭代器类似。

插入，删除操作使用的是statement.executeUpdate(String sql)

类似这样 

   try {
       statement = connection.createStatement();
       System.out.print("INSERT RESULT :" + statement.executeUpdate("insert into test(name,id) values ('123' , 7)")+  "\n");
        } catch (SQLException e) {
            e.printStackTrace();

    }

5， 当不使用数据库时，使用Connection.close关闭数据库连接

三个主要语句的总结：

statement.execute(String sql)

执行sql语句，如果是查询语句（返回ResultSet），那么返回值是true，否则返回值是false

statement.executeUpdate（String sql）

执行sql语句，增、删、改使用，返回的是影响的行数

statement.executeQuery(String sql)

执行sql语句，执行查询操作，返回的是ResultSet

Java泛型相关

泛型这个东西，确实是每天都在用，但是却我真的没有仔细好好深入的部分

ArrayList list = new ArrayList<String>();

这就是一个典型的泛型使用，我们创建了一个ArrayList容器，容纳的是String

泛型是JDK5之后提供的语言特性，创造出来主要是为了提高代码的复用率

泛型这个更多的是用在“容器”上

public static class geneic<T>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
} 
public static void main(String[] args)
{
   new geneic<Integer>(2).out();
   new geneic<Double>(2.2).out();

}

这就是我们创建的一个泛型示例，T的意思是type，创建两个generic容器，把一个类型容纳进去。
输出的结果是

java.lang.Integer

java.lang.Double

泛型用于创建各种类型的“容器”，在创建的时候，我们还可以加以限制

1.extends

用法：

public static class geneic<T extends Number>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
}

extends，继承自某个类，这里的T extends Number 把能加载到这个容器的类限制到了Number的子类，那就是说new geneic<Integer>(2)是可以的，但是new geneic<Integer>("123")是不可以的

2,super

用法基本和extends相同，一样是用来限制容器可以加载的类

public static class geneic<T extends Number>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
}

super本身就是“超类”的意思，那么也就是把这个容易可以加载的部分限制了一个最小下界T，T以及T的父类都可以加载到这个容器之中

泛型的一些使用：

我们定义三个类：

Fruit

Apple

RedApple

继承关系从名字上都可以看出来

有一点需要注意的是，Java创建对象的时候是不能使用通配符的，List<? extends Fruit> fruitList = new ArrayList<Fruit>();是可以的，但是List<? extends Fruit> fruitList = new ArrayList<?>();则会报错

对于ArrayList来说，里面只能存放一种类型

我们这样子来使用泛型：

List<? extends Fruit> fruitList = new ArrayList<Fruit>();

fruitList.add(new Fruit()); //报错
fruitList.add(new Apple()); //报错

这么写会编译错误，原因就在于，这里? extends Fruit意味着向里面放入可能是Fruit也可能是某种子类，编译器没有办法保证其安全

我们换一种方式：

List<? super Fruit> fruitList = new ArrayList<Fruit>();

fruitList.add(new Apple());
fruitList.add(new Fruit());

这样子写不会有编译错误，子类cast成父类是不会有问题的

还有我们一些要在方法中注意的问题

上界声明：

extends可以get，但是不能add

 public void upperBound(List<? extends Fruit> list,Fruit fruit)
{
    list.get(0); //get没有问题
    list.add(fruit); //add这一句报错
}

原理和上边一样，编译器不能保证类型安全

下界声明：

super可以add，但是不能get

 public void lowerBound(List<? super Fruit> list,Fruit fruit)
{
    list.get(0); //get会报错
    list.add(fruit); //add没有问题

}

原理也是同上，List这种list里面添加一个fruit/apple是没有问题的，但是返回的时候就没办法说是哪一种类型，不知道是Object还是Fruit，所以报错
