Connect to Evernote - 印象笔记 Android SDK 使用日记 （二） 初步使用

我们导入官方的Demo后，大概可以看到目录是这个样子的：

这里面，library是依赖部分，核心的业务处理，如果我们没有额外的要求，这部分就尽管引用好了，不要管他。

Demo运行HelloEDAM之后就会出现默认启动Activity

如果是自己建立依赖印象笔记SDK的工程，我们就把library设定为一个依赖的Module即可。

初始设定,建立Session

HelloEDAM这个Activity是集成自ParentActivity的，所以在HelloEDAM中，有些变量需要到ParentActvity中找一下

在ParentaActivity开头，我们可以找到这么一段代码，这里我们需要输入我们自己的API key和secret才能正常使用。

// Your Evernote API key. See http://dev.evernote.com/documentation/cloud/
// Please obfuscate your code to help keep these values secret.
private static final String CONSUMER_KEY = "****";
private static final String CONSUMER_SECRET = "****";

我们需要声明一个EvernoteSession变量，并取得它的实例，我们会使用这个对象建立和印象笔记的连接

 EvernoteSession mEvernoteSession;
 mEvernoteSession = EvernoteSession.getInstance(this, CONSUMER_KEY, CONSUMER_SECRET, EVERNOTE_SERVICE, SUPPORT_APP_LINKED_NOTEBOOKS);

EVERNOTE_SERVICE是一个 EvernoteSession.EvernoteService 类型，指Evernote的服务，我们现在暂时只能使用沙箱服务，所以，只能这样子定义 

 EvernoteSession.EvernoteService EVERNOTE_SERVICE = EvernoteSession.EvernoteService.SANDBOX;

SUPPORT_APP_NOTEBOOKS是一个boolean对象，true代表只能连接一个笔记本，而false代表可以连接多个

注意，Demo的这些变量都是定义在ParentActivity里面的

还有，注意在manifest里面需要添加一个Activity，为后面登入的Activity做注册（这个Activity存在于依赖包里面，这里不注册后面没办法登录了）

  <activity android:name="com.evernote.client.android.EvernoteOAuthActivity" android:configChanges="orientation|keyboardHidden" />

当然，还不要忘记添加连网权限

<uses-permission android:name="android.permission.INTERNET" ></uses-permission>

Log in/out 登入登出模块，Oauth验证

我以为印象笔记是类似微博的那种SSO的方式…结果不是那样子的，他是打开一个网页进行Oauth授权
SSO和Oauth这两种方式，各有千秋吧。

取得EvernoteSession的实例之后，ß我们就可以用我们之前在沙盒里面注册的帐号了

mEvernoteSession.authenticate(this);

这一句话就可以进行Evernote的Oauth认证

登出同样是一句话

mEvernoteSession.logOut(this);

如果要验证的话，那么可以使用下一句话验证是否登录进去

mEvernoteSession.isLoggedIn()

UserStore 获取用户信息

我们在经过Oauth认证之后可以使用UserStore方式获取用户信息

这里官网上也没有做更详细的解释，基本都是摸索着来，而且虽然官网说是用UserStore，更详细的应该说是使用AsyncUserStoreClient，请看以下代码

 /* 如果已经登录成功了 */
    if (mEvernoteSession.isLoggedIn())
    {
        AsyncUserStoreClient mUserStoreClient = null;
        try {
            /* 获取AsyncUserStoreClient对象 */
            mUserStoreClient = mEvernoteSession.getClientFactory().createUserStoreClient();
            /* 异步获取用户信息 */
            mUserStoreClient.getUser(new OnClientCallback<User>() {
                @Override
                public void onSuccess(User data) {
                    Log.i(TAG,data.toString());
                    Toast.makeText(MainActivity.this,data.getUsername() + " 已绑定成功",Toast.LENGTH_LONG).show();
                }

                @Override
                public void onException(Exception exception) {

                }
            });
        } catch (TTransportException e) {
            e.printStackTrace();
        }

    }

data打印出来会是这个样子

  User(id:xxxxx, username:xxxx, privilege:NORMAL, active:true, shardId:s1, accounting:Accounting(uploadLimit:62914560, uploadLimitEnd:1403766000000, uploadLimitNextMonth:62914560, premiumServiceStatus:NONE))

选择我们感兴趣的地方get即可

NoteStore 写入笔记信息

写入笔记信息关键是使用Note这个类作为内容提供，同时我们需要了解一些ENML（印象笔记的格式）的东西，使用AsyncNoteStoreClient进行写入

Note可以setTitle和setContent，我们可以非常方便的进行数据的写入。

使用ENML格式，需要加一些文件头，Evernote在它的Util里面有提供。

AsyncNoteStoreClient的使用方式和AsyncUserStoreClient的使用方法很像。

  public void writeNote(View v)
{
     EditText titleEditText = (EditText)findViewById(R.id.editText_title);
     EditText contentEditText = (EditText)findViewById(R.id.editText_content);

    try {
        createNote(titleEditText.getText().toString(),contentEditText.getText().toString());
    } catch (TTransportException e) {
        e.printStackTrace();
    }
}
public void createNote(String title, String content) throws TTransportException {
    if (mEvernoteSession.isLoggedIn()) {
        Note note = new Note();
        note.setTitle(title);
        note.setContent(EvernoteUtil.NOTE_PREFIX + content + EvernoteUtil.NOTE_SUFFIX);
        mEvernoteSession.getClientFactory().createNoteStoreClient().createNote(note, new OnClientCallback<Note>() {
            @Override
            public void onSuccess(final Note data) {
                Toast.makeText(getApplicationContext(), data.getTitle() + " has been created", Toast.LENGTH_LONG).show();
            }

            @Override
            public void onException(Exception exception) {
                Log.e(TAG, "Error creating note", exception);
            }
        });
    }
}

总结

读取用户信息，写入简单笔记，对于Evernote的SDK来说，非常简单，在经历了沙箱机制不爽之后，使用Evernote 的android SDK真是赏心悦目，另外他的异步方法封装的很好，使用的时候都不需要考虑异步调用的问题

另外，关于SDK的使用方法，印象笔记的dev站点还不如Github主页Readme的清楚…所以，还是看Github吧。
