算法-动态规划相关

动态规划是一种很长见的算法，经典的题目如LCS，LIS都会用到动态规划。

动态规划的特点是，动态规划会存储上一个状态的值，用递推公式去推下一个状态，保存起来，最终来得到最优解。

我们需要找到某个状态的最优解，推下去得到其他状态的最优解。

我们来通过两个经典的背包问题来说明动态规划：

0-1背包问题

There are 5 items that have a value and weight list below, the knapsack can contain at most 100 Lbs

values: 20 30 65 40 60

weight: 10 20 30 40 50

Solve the problem as 0/1 knapsack.

0-1背包问题，就是有一个容量是V的背包，第i件的重量是w[i],价值是v[i],每个物品只能装一次，那么如何让背包能装的价值最大

理论上这个问题也可以用贪心去解决，但是贪心不一定能得出来合适的解，我们使用动态规划来解决这个问题

0-1背包，指的就是对于一个物品只有两个选择：放，或者不放

动态规划需要明确两个方面：状态和状态转移方程

我们使用f[i][v]表示前i个物品放入容量为v的背包可以获得的最大价值
f[i][v] 就是一个状态

状态转移方程是：

f[i][v] = max{f[i-1][v]，f[i-1][v-w[i]]+v[i]}

f[i-1][v] 就是这个物品不放的情况下状态

f[i-1][v-w[i]] + w[i] 就是在放这个物品的情况下的状态

贴上代码，详情在注释中有体现：

public class Main {
/* 容量 */
public static int V = 100;
/* 物品总数 */
public static int I = 5;
/* 各个物品的values */
int[] values = {20,30,65,40,60};
/* 各个物品的weight */
int[] weight = {10,20,30,40,50};
/* 建立储存状态的二维数组 ，表示在V容量装前I个物品的价值，加了1是为了让数组下标I就是实际的件数，V就是实际的容量*/
int[][] f = new int[I+1][V+1];

public int knapSack()
{
    /* i代表装物品的件数，我们可以直接从1开始循环，因为i=0的时候没有意义（一个物品都不装，肯定价值是0）*/
    for (int i=1;i<=I;i++)
    {
        /* j代表容量，容量是0的时候也没有意义，所以直接从1开始循环 */
        for (int j=1;j<=V;j++) {
                /*如果我没有装的下这个物品的容量，那么和之前的价值是一样的*/
                if (j < weight[i-1])
                    f[i][j] = f[i][j - 1];
                else
                    /*我有了装的下这个物品的容量，那么我们就上我们的递归公式，在装和不装中取一个和谐的值 */
                    f[i][j] = max(f[i - 1][j], f[i - 1][j - weight[i-1]] + values[i-1]);
                /*测试输出每次的i,j,f */
                System.out.print("i:" + i + " j:" + j + " f:" + f[i][j] + "\n");

        }
    }
    /* 返回最大值 */
    return f[I][V];
}
/* 取两个数大的那一个 */
public int max(int a,int b)
{
    if (a< b )
        return b;
    else
        return a;
}

public static void main(String[] args)
{
    Main main = new Main();
    System.out.print(main.knapSack());

}
}
