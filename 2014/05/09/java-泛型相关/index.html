Java泛型相关

泛型这个东西，确实是每天都在用，但是却我真的没有仔细好好深入的部分

ArrayList list = new ArrayList<String>();

这就是一个典型的泛型使用，我们创建了一个ArrayList容器，容纳的是String

泛型是JDK5之后提供的语言特性，创造出来主要是为了提高代码的复用率

泛型这个更多的是用在“容器”上

public static class geneic<T>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
} 
public static void main(String[] args)
{
   new geneic<Integer>(2).out();
   new geneic<Double>(2.2).out();

}

这就是我们创建的一个泛型示例，T的意思是type，创建两个generic容器，把一个类型容纳进去。
输出的结果是

java.lang.Integer

java.lang.Double

泛型用于创建各种类型的“容器”，在创建的时候，我们还可以加以限制

1.extends

用法：

public static class geneic<T extends Number>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
}

extends，继承自某个类，这里的T extends Number 把能加载到这个容器的类限制到了Number的子类，那就是说new geneic<Integer>(2)是可以的，但是new geneic<Integer>("123")是不可以的

2,super

用法基本和extends相同，一样是用来限制容器可以加载的类

public static class geneic<T extends Number>
{
    public T var;
    public geneic(T c)
    {
        var = c;
    }
    public void out()
    {
        System.out.print(var.getClass().getName() + "\n");
    }
}

super本身就是“超类”的意思，那么也就是把这个容易可以加载的部分限制了一个最小下界T，T以及T的父类都可以加载到这个容器之中

泛型的一些使用：

我们定义三个类：

Fruit

Apple

RedApple

继承关系从名字上都可以看出来

有一点需要注意的是，Java创建对象的时候是不能使用通配符的，List<? extends Fruit> fruitList = new ArrayList<Fruit>();是可以的，但是List<? extends Fruit> fruitList = new ArrayList<?>();则会报错

对于ArrayList来说，里面只能存放一种类型

我们这样子来使用泛型：

List<? extends Fruit> fruitList = new ArrayList<Fruit>();

fruitList.add(new Fruit()); //报错
fruitList.add(new Apple()); //报错

这么写会编译错误，原因就在于，这里? extends Fruit意味着向里面放入可能是Fruit也可能是某种子类，编译器没有办法保证其安全

我们换一种方式：

List<? super Fruit> fruitList = new ArrayList<Fruit>();

fruitList.add(new Apple());
fruitList.add(new Fruit());

这样子写不会有编译错误，子类cast成父类是不会有问题的

还有我们一些要在方法中注意的问题

上界声明：

extends可以get，但是不能add

 public void upperBound(List<? extends Fruit> list,Fruit fruit)
{
    list.get(0); //get没有问题
    list.add(fruit); //add这一句报错
}

原理和上边一样，编译器不能保证类型安全

下界声明：

super可以add，但是不能get

 public void lowerBound(List<? super Fruit> list,Fruit fruit)
{
    list.get(0); //get会报错
    list.add(fruit); //add没有问题

}

原理也是同上，List这种list里面添加一个fruit/apple是没有问题的，但是返回的时候就没办法说是哪一种类型，不知道是Object还是Fruit，所以报错
