Connect to Evernote - 印象笔记 Android SDK 使用日记 （一）

转投到印象笔记

印象笔记是个好东西呀，以前我用的一直是为知笔记，主要是因为Windows下客户端表现尚可，原生支持markdown，上传流量给的很大（相对于印象笔记），速度也比较快，移动客户端做的也还行…

但是，但是，我在用mac的时候，对比一下mac版本的印象笔记和为知笔记，为知笔记完全被比下去了，可能也和国内外mac的使用率有关系吧，印象笔记mac版简直比windows版还要优秀，并且现在有了马克飞象这种神器，用markdown写印象笔记毫无压力，所以，我义无反顾地转投到了印象笔记的怀抱。

我是用的是国内版的印象笔记，感觉印象笔记比Evernote International的同步速度能快不少，功能上现在基本也没什么差别了。

获取印象笔记API Key

印象笔记是个好工具，它开放了相关的API，可以让我们Connect to Evernote。

首先我们需要到 http://dev.yinxiang.com/doc/ 来申请一个API Key，申请的时候，注意印象笔记是有两种API key的，分为 基本访问权限 和 完整访问权限，区别可以点这里http://dev.yinxiang.com/doc/articles/permissions.php 查看

简而言之，基本访问权限只能写入笔记，而完全访问权限可以读取以前的笔记，还可以写入笔记。

那么，我们就直接选用一个完全访问权限就好了？这里还是得按需求，因为正式上线的时候才需要印象笔记的审核，你需要给出使用完全访问权限的理由，否则，不予通过。

印象笔记通过一个沙箱(sandbox)机制，使得如果你这个app没有通过审核的话，只能在印象笔记的沙箱服务器(sandbox.evernote.com)使用，而不能连接到真正的生产服务器。

另外…我不知道为什么，申请API的时候他说会给我的邮箱里面发送一个含有API key和API secret的邮件，但是我申请了两个，我的邮箱都没收到这个邮件……所以，记得保存API key和API secert。

关于印象笔记API的沙箱机制

这个沙箱机制，也许设计的初衷是好的，使调试不会影响到实际的服务器，但是，最终，这个机制让开发者多多少少会遇到一些麻烦。

微博对API的控制方式是，我可以添加测试帐号，我只能有几个测试帐号可以做API的测试，审核通过后，非测试帐号也可以进行API的操作了。但是印象笔记呢，要求你直接在一个沙箱服务器里面创建一个新的帐号，这个帐号和印象笔记，Evernote International都不相关，但是沙箱里面任何一个帐号都可以使用你的API。

好了，也许和上述和微博的对比，还没发现“麻烦“所在，麻烦的就在于，你在使用印象笔记的时候，见过“登录到印象笔记”，“登录到Evernote International”的选项，你见过“登录到沙箱”这个选项么…

所以，需要更改一些设置才能登录：

首先我们在沙箱中注册：
https://sandbox.evernote.com/Registration.action

接下来，我们需要对客户端进行一些修改：

对于Windows客户端：

  1. 在“文件”菜单中退出印象笔记。如果出现提示框，就确认你需要完全退出印象笔记。
  2. 运行 regedit 注册表编辑器
  3. 打开 HKEY_CURRENT_USER\Software\Evernote 键
  4. 修改 SyncServiceUrl 的值，把 app.yinxiang.com 改为 sandbox.evernote.com
  5. 退出 regedit 注册表编辑器
  6. 打开印象笔记，并登陆你的沙箱帐户，注意：你需要在你的沙箱账户的用户名后面加上”#sandbox.evernote.com” （不包含引号）

对于Android客户端（待验证）

  1. 关闭印象笔记应用
  2. 复制这个引导文件到你的 SD 中的 “/sdcard/Evernote” 目录，这个引导文件应该是需要被命名为bootstrap
  3. 重启印象笔记应用

对于其他移动客户端（非Android）

注销当前的印象笔记帐户。在登陆界面，输入你的沙箱用户名和密码，并在用户名后加上 “#sandbox.evernote.com”，然后登陆。例如，要使用沙箱帐号 “foo” 和密码 “bar” 登陆, 你需要这样输入：

用户名： foo#sandbox.evernote.com

密码： bar

官方文档在此：http://dev.yinxiang.com/doc/articles/testing.php

我对官网上面的文档做了一些简单的修改，修改的部分是粗体字，因为…我一开始按照官网的指示就没成功啊摔！！！

另外，没有mac版？

印象笔记你mac版做的这么好，没说怎么登录沙箱？

我在Evernote开发者论坛上找到一个Evernote Employee说的一个在mac上登录沙箱环境的回复，我没有亲身试过，大家需要的可以试试看

  1. Quit Evernote from the elephant icon in your menu bar.
    Open a terminal.
  2. Move your existing Evernote data directory:
    mv ~/Library/Application\ Support/Evernote ~/Library/Application\ Support/Evernote.production
  3. Change the API endpoings by running the three commands below:

    defaults delete com.evernote.Evernote
    defaults write com.evernote.Evernote noteStoreURL http://sandbox.evernote.com/edam/note
    defaults write com.evernote.Evernote userStoreURL https://sandbox.evernote.com/edam/user

  4. Open Evernote and log in to your sandbox account

OK,我们这就算搞定了蛋疼的沙箱环境。

下载印象笔记Android SDK

这是印象笔记 Android SDK 项目托管的地址：
https://github.com/evernote/evernote-sdk-Android

我们按照Readme里面进行项目导入即可。我用的IDE是Idea，直接导入project没有遇到什么问题，运行Simple的时候，注意选择HelloEDAM而不是Simple Note。

一些总结

略蛋疼的沙箱机制，让我登录就搞了一晚上（因为Android死活登录不进去……），Evernote的文档资料，中文的部分不算特别全，尤其是Android，有些东西还是看Simple和SDK Github里面的Readme吧…

其实，要是仅仅做分享的话，没必要使用印象笔记SDK，只需要使用ShareSDK来做就好了，会比使用印象笔记简单很多，顺带还能分享到更多的平台。如果说对印象笔记有其他的需求，那么我们还是继续来深入研究一下吧。
