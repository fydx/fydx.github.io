title: iOS CoreData基本使用
date: 2015-01-19 23:31:13

tags:iOS

CoreData是一个在iOS下做持久层数据存储非常好用的方法，整个过程中无需SQL语句。

创建项目

我创建一个名为“StoreSystem”的项目，注意下面要勾选“Use Core Data“

创建Model

假设我们有三个数据Model类，Store(商场)，Department（部门），Staff（员工），具体属性如下图：

我们在项目文件夹里面可以找到.xcdatamodeld的文件，点击进入后，我们创建Model的工作主要会在这里执行。

在Xcode的下面我们可以找到“Add Entity”

点击之后，在ENTITIES中会出现我们刚刚添加的Entity，可以添加属性，关系等等。我们在这里把三个Model都作为实体添加进去。

Attributes很简单，设置属性名字，类型，点点鼠标就行。

RelationShip可以定义实体之间的关系，比如部门是属于商店的，这是一个多对一的关系，可以在Store和Department的RelationShip中各创建一个关系，Store中Destination选择Department，Department中的Destination中选择Store，inverse选择对方实体的关系，这样子创建了一个双方的关系。

可以在Xcode右边的选项卡中，在Store上创建的关系中选择 To Many，这样我们就创建了一个一对多的关系。

同理，对Staff和Department再做一次添加一对多关系的处理。

在Xcode下面的Editor Style中可以选择另外一种视图样式，在一个实体上面按Control，往另一个实体中拖，即可创建一个双方的关系，不过一对多还是需要手动去添加的。

添加完关系之后，切换Editor Style，看到的图大概是这个样子的：

OK，基本Model的创建基本结束。

创建数据Model类

在xcdatamodeld文件中我们创建好了数据模型之后，Xcode可以根据这个文件直接生成Objetctive-C的h和m文件

我们选择File—New—File，选择iOS—Core Data 中的NSManagedObject subclass，生成我们所需的h和m文件

OK，接下来要做的就是实现增删改查这些数据库的基本功能了。

实现Store的“增“

用Storyboard创建了一个简单的界面，按“确认添加”按钮，即为增加。

核心“增”的方法如下：

- (void)saveStore
{
AppDelegate *appDelegate = [UIApplication sharedApplication].delegate;
NSManagedObjectContext *managedContext = [appDelegate managedObjectContext];
if (managedContext != nil) {
    if (_store == nil) {
        _store = [NSEntityDescription insertNewObjectForEntityForName:@"Store" inManagedObjectContext:managedContext];
        if (_nextId) {
             _store.storeId = @(_nextId);
        }

    }
    _store.name  = _nameTextField.text;
    _store.address = _addressTextField.text;
    _store.number = _numberTextField.text;
    _store.tel = _telTextField.text;
}
[appDelegate saveContext];
[self.navigationController popViewControllerAnimated:YES];

}

这里面，需要注意的是

  • 从AppDelegate中获取managedObjectContext
  • 用NSEntityDescription去获取要插入的对象（不能直接alloc init）
  • saveContext，储存更改

读取数据

- (void)loadStores
{
AppDelegate *appDelegate = [UIApplication sharedApplication].delegate;
NSManagedObjectContext *managedContext = appDelegate.managedObjectContext;
//指定数据表
NSEntityDescription *entity = [NSEntityDescription entityForName:@"Store" inManagedObjectContext:managedContext];
//新建一个请求命令，相当于select
NSFetchRequest *request = [[NSFetchRequest alloc]init];
//request 指定数据表
[request setEntity:entity];
//错误描述
NSError *error;
//接收查询出来的数据
NSArray *arr = [managedContext executeFetchRequest:request error:&error];
_stores = [arr mutableCopy];

}

在获取managedContext后，获取指定数据表，执行查询，获得全部的store。
这里就是把所有的数据都读取出来了，我们对数据还可以进行筛选，再NSFetchRequest这里做文章，这里先不展开了。

删除数据

- (void)deleteStoreAtIndex:(NSUInteger)index
{
AppDelegate *appDelegate = [UIApplication sharedApplication].delegate;
NSManagedObjectContext *managedContext = [appDelegate managedObjectContext];
[managedContext deleteObject:_stores[index]];
NSError* error = nil;
[managedContext save:&error];

}

这个也很简单，调用managedContext的deleteObject方法即可。

修改数据

这是CoreData非常爽的一个部分，我们在获取到一个store对象之后，直接对这个对象的属性进行修改，修改完毕后执行[managedContext save] 即可，我们的修改就持久化了。

Department和Staff的设定也大致相同。

关于relationship

我们在生成完了Store，Department，Staff这三个类之后，在Store和Department的h文件可以看到有几个多出来的方法，以Store类为例，再Store.h里面可以看到：

- (void)addDepartmentsObject:(Department *)value;
- (void)removeDepartmentsObject:(Department *)value;
- (void)addDepartments:(NSSet *)values;
- (void)removeDepartments:(NSSet *)values;

因为有Relationship的存在，在Store里面还有一个属性departments，我们再创建Department的时候，调用[store addDepartmentObject:xxxx] ，即可把我们创建的Department和Store生成出来一条关系，我们的department便被加到Store的departments中。

relationship是帮助我们来控制各个Entity之间的关系，有了它会更方便，但是也可以自己来通过其他的方式，比如用类似数据库的Foreign key等等的方式来控制。

总结

iOS的CoreData框架使用实在是太方便了，相较于android各类ORM框架来说，由于有Apple加上的一层封装，使用起来实在是爽的飞起。但是由于Apple的的封闭框架，像自增id这种事情都没办法去做，自由度相对于android的ORM框架欠缺不少。

总之，使用CoreData可以解决绝大多数的数据存储问题。

Android sqlite 操作（单例sqlite helper）

Android的数据库使用的是sqlite，sqlite仅由一个单文件组成，非常非常简单，备份起来，也就是一个.db文件的事儿。

我们希望的一个方式是，使用单例去控制数据库，增删改查均可在一个单例中去实现。

Android为我们提供了一个框架，叫SQLiteOpenHepler，我们可以继承这个抽象类，实现我们想要的功能。

获取数据库实例

想要实现单例的数据库访问和增删改查，核心就是获取数据库实例，也就是SQliteDatabase，获取了数据库就可以执行各种sql语句，那么数据库就可以任意操作了，下面代码mDB就是我们获取到的数据库，增删改查这些操作，使用mDB.exec(sql)即可。

getWritableDatabase以读写方式获取的数据库，一旦数据库的空间满了，数据库就无法打开，抛出异常
getReadableDatabase也是以读写方式打开数据库，但是一旦数据库空间满了，会再尝试以只读的方式打开数据库

public class DBHelper extends SQLiteOpenHelper {
/* 静态实例 */
public static DBHelper mInstance = null;
/* 数据库名字 */
public static final String DB_NAME = "notes.db";
/* 数据库版本，为onUpgrade准备 ，这个数字必须大于等于1 */
public static final int DB_VERSION = 1 ;
/* 数据库实例 */
private SQLiteDatabase mDB = null;
/* context 上下文 */
private static Context mContext = null;

public static String CREATE_TABLE_NOTES = "CREATE TABLE IF NOT EXISTS notes (_id INTEGER PRIMARY KEY AUTOINCREMENT,  content TEXT,)";

/**
 * 静态构造方法
 * @throws SQLiteException
 */
private DBHelper() throws SQLiteException
{
    /* 参数：1，上下文Context，2，数据库名字，3，CursorFactoty，可以自定义Cursor的创建
            传入null就是以默认状态，4，数据库版本 */
    super(mContext,DB_NAME,null,DB_VERSION);
    getDB();

}

/**
 * 获取Database实例
 */
public void getDB()
{
    try
    {
        mDB = getWritableDatabase();
    }
    catch (Exception e)
    {
        Log.e("Database","GET DB ERROR!! " + e.toString());
        mDB = getReadableDatabase();
    }
    Log.i("Dabatase","GET DB SUCCESS!!");
}    /**
 以单例的方式获取对象
 */
public static synchronized  DBHelper getInstance(Context context)
{
    mContext = context;
    if (mInstance == null)
    {
        mInstance = new DBHelper();

    }
    return mInstance;

}

/**
 * 数据库被首次创建时执行的方法
 * @param sqLiteDatabase
 */
@Override
public void onCreate(SQLiteDatabase sqLiteDatabase) {

    sqLiteDatabase.execSQL(CREATE_TABLE_NOTES);
}

/**
 * 数据库传入的版本号与当前数据库版本号不同时执行的方法
 * @param sqLiteDatabase
 * @param i
 * @param i2
 */
@Override
public void onUpgrade(SQLiteDatabase sqLiteDatabase, int i, int i2) {
    String sql = "DROP TABLE  IF EXISTS notes" ; 
    sqLiteDatabase.execSQL(sql);
    sqLiteDatabase.execSQL(CREATE_TABLE_NOTES);
}
}

增删改查动作的一些执行方法

我们操作的是数据库，那么必然可以使用sql语句执行增删改查的动作，但是写sql这种事情需要拼接字符串，毕竟不方便。对于在Android中的sqlite，我们还会有一些另外的方法

增：

使用ContentValues，作为一个键值对。

values.put("column","value") 第一个参数是列名，第二个参数是值，根据列的数值类型而定，可以传入基本数据类型或String

put结束之后，调用db.insert即可，

insert的参数是这样子的 public long insert (String table, String nullColumnHack, ContentValues values)

第一个参数是表名
第二个参数是为了避免values是null，如果sql语句的values有值但是前面的列名是null的时候，会给一个默认的列，避免sql语句不合法。ContentValues的put方法，第一个参数是允许为空的，一旦第一个参数是空的，那么就会给它分配到nullColumnHack这个列上面去
第三个参数就是ContentValues

返回值是新插入数据的Row ID

 public void insertNotes(LocalNote localNote)
  {

    ContentValues values = new ContentValues();
    values.put("content",localNote.getContent());
    mDB.insert("notes",null,values);
    Log.i("Database","INSERT SUCCESS : " + values.toString());
}

改:

和增一样，不过改的时候多了一个where

public int update (String table, ContentValues values, String whereClause, String[] whereArgs)

第一个第二个参数和在增里面的是一样的
第三个参数，选择进行哪些数据的更新，其实也就是sql语句里面where之后的东西。这里要是传入的是null会进行所有数据的更新
第四个参数，替换第三个参数里面的 “?”

返回值是影响的列数

public void updateNotes (LocalNote localNote,int id)

第三个参数，替换第二个参数里面的 ?

不得不说这个API的设计…略蛋疼

public Cursor query (boolean distinct, String table, String[] columns, String selection, String[] selectionArgs, String groupBy, String having, String orderBy, String limit, CancellationSignal cancellationSignal)

10个参数的方法…

  1. 是否去除重复数据，是的话传true，不是的话传false

我们需要声明一个EvernoteSession变量，并取得它的实例，我们会使用这个对象建立和印象笔记的连接

Connect to Evernote - 印象笔记 Android SDK 使用日记 （一）

转投到印象笔记

印象笔记是个好东西呀，以前我用的一直是为知笔记，主要是因为Windows下客户端表现尚可，原生支持markdown，上传流量给的很大（相对于印象笔记），速度也比较快，移动客户端做的也还行…

但是，但是，我在用mac的时候，对比一下mac版本的印象笔记和为知笔记，为知笔记完全被比下去了，可能也和国内外mac的使用率有关系吧，印象笔记mac版简直比windows版还要优秀，并且现在有了马克飞象这种神器，用markdown写印象笔记毫无压力，所以，我义无反顾地转投到了印象笔记的怀抱。

我是用的是国内版的印象笔记，感觉印象笔记比Evernote International的同步速度能快不少，功能上现在基本也没什么差别了。

获取印象笔记API Key

印象笔记是个好工具，它开放了相关的API，可以让我们Connect to Evernote。

首先我们需要到 http://dev.yinxiang.com/doc/ 来申请一个API Key，申请的时候，注意印象笔记是有两种API key的，分为 基本访问权限 和 完整访问权限，区别可以点这里http://dev.yinxiang.com/doc/articles/permissions.php 查看

简而言之，基本访问权限只能写入笔记，而完全访问权限可以读取以前的笔记，还可以写入笔记。

那么，我们就直接选用一个完全访问权限就好了？这里还是得按需求，因为正式上线的时候才需要印象笔记的审核，你需要给出使用完全访问权限的理由，否则，不予通过。

/**

public class ComingCallReceiver extends BroadcastReceiver {

private final String TAG = "PhoneState";
private static String phoneNumber = null;
@Override
public void onReceive(Context context, Intent intent) {
    String action = intent.getAction();
    if(intent.getAction().equals(Intent.ACTION_NEW_OUTGOING_CALL)){
        //拨打电话的状况
    }else{
        //android没有接收来电的action，所以，不是拨打电话，就是接收来电
        doReceivePhone(context, intent);
    }

}

/**
 * 处理来电
 * @param context
 * @param intent
 */
public void doReceivePhone(Context context, Intent intent) {

    TelephonyManager telephony =
            (TelephonyManager)context.getSystemService(Context.TELEPHONY_SERVICE);
    int state = telephony.getCallState();
    switch(state){
        case TelephonyManager.CALL_STATE_RINGING:
            Log.i(TAG, "来电 ："+phoneNumber);
            /* 这里只有在来电的时候才能获取到PhoneNumber，其他的时候用这种方式获取的是null，
                所以PhoneNumber使用一个静态变量来保存 */
            phoneNumber = intent.getStringExtra(
                    TelephonyManager.EXTRA_INCOMING_NUMBER);
            break;
        case TelephonyManager.CALL_STATE_IDLE:
            Log.i(TAG, "挂断 ："+phoneNumber);
            break;
        case TelephonyManager.CALL_STATE_OFFHOOK:
            Log.i(TAG, "通话 ："+phoneNumber);
            break;
    }
}
}

LCS最长公共子串

LCS（最长连续公共子串）算法

假设我们有两个字符串 ABCDMARK1234 ， MARKDOWN34

那么最长公共子串就是MARK

我们建立假设原来的子串分别为A，B，我们建立一个新的数组C，去记录最长的连续子串的位置。

递推公式：
C[i,j] = 0 (A[i]不等于B[j])
= 1 (A[i]等于B[i]且i或j小于1)
= C[i-1,j-1] + 1 (A[i]等于B[j])

使用m，n记录C数组中最大的位置

public class Main {

public static String A = "ABCDMARK1234" ;
public static String B = "MARKDOWN" ;
public static int[][] C ;
public static int m;
public static int n;
public static int maxValue = 0;
public static void main(String[] args)
{
    /* 创建二维数组C */
    C = new int[A.length()][B.length()];
    /* i,j循环 */
    for (int i = 0 ; i<A.length();i++)
        for (int j =0 ; j<B.length(); j++)
        {
            if (A.charAt(i) == B.charAt(j)) {
                /* 如果i或者j小于1，那么当前的值直接置为1 */
                if (i<1 || j <1)
                    C[i][j] = 1;
                else{
                    C[i][j] = C[i - 1][j - 1] + 1;
                }

                if (C[i][j] > maxValue)
                {
                    maxValue = C[i][j];
                    m = i;
                    n = j;
                }
            }
            else
                C[i][j] = 0;

        }
    /* 将m和n指向匹配第一个字符的位置 */
    m = m - maxValue + 1 ;
    n = n - maxValue + 1 ;
    /* 循环输出 */
    for (int i=0;i<maxValue;i++)
    {
        System.out.print(A.charAt(m));
        m++;
    }
}
}

算法-矩阵链乘积

矩阵链问题描述：

给定n个矩阵构成的一个链，其中i=1,2,…n，矩阵A的维数为pi-1 pi ，对乘积 A1 A2 …An 以一种最小化标量乘法次数的方式进行加全部括号。

括号其乘积的顺序是会影响到需计算乘积所需简单算术运算的数目，即其效率。例如，设A为一10×30矩阵，B为30×5矩阵与C为5×60矩阵，则
(AB)C 有 (10×30×5) + (10×5×60) = 1500 + 3000 = 4500 个运算
A(BC) 有 (30×5×60) + (10×30×60) = 9000 + 18000 = 27000 个运算

我们的目标就是找出来矩阵相乘的最佳顺序,得到最小的相乘次数

对于矩阵链问题，最好的解决方法是动态规划。

首先我们来找一下最优子结构：

我们需要计算整个矩阵链的的最优解，计算过程中，肯定有一个k（1 这两部分，寻找这两部分的最优解。

再具体一点，最优子结构就是指：矩阵链的最优解如果是在k(i和也是最优解。我们可以根据把问题分割，以构建总问题的最优解。

我们再来寻找状态和状态转移方程：

我们确定一个状态m[i,j] ,这指的是计算矩阵链 所需要的乘法次数的最小数目。

我们在按照之前的思路进行分解，以k做左右两边的分解，我们就可以得到递归式：

m[i,j] = 0 (i=j)
= min(i<=k<j) {m[i,k]+m[k+1,j]+pi-1pkpj} (i<j)

我们把k从i到j都试一次，取最小值。

这个时候，我们如果使用递归求解，时间复杂度会变为指数级，我们为了降低时间复杂度，可以再来开一个数组s[i][j]记录矩阵链取得最优时k的值。

代码如下，关于代码说明写在代码注释中：

/**
 * Created by libin on 14-5-15.
 */
public class Main {
/* 这里虽然是5个数字，但是实际上是4个矩阵 */
public static int Data[] = {3,5,2,1,10};
public static void main(String[] args)
{
  System.out.print("Result :" +  maxChain(Data));
}
public static int maxChain(int[] A)
{
    /* m 对应保存从Ai到Aj的最小的乘的次数 也就是从第i个矩阵到第j个矩阵，所以A[0]的元素都是没用的 */
    int[][] m = new int[A.length][A.length];
    /* s 对应保存从Ai到Aj的最佳的k ，和m一样，数组第一行是无效的*/
    int[][] s = new int[A.length][A.length];
    /* 设定一个大数，为了之前把数组m里面所有的值标记为无限大 */
    int MAX = 10000000;
    /* n是数组最大下标，这个值可以省略掉 */
    int n = A.length-1;
    /* 我们事前首先把所有的数字置成0 */
    for(int i=1;i<n+1;i++){
        m[i][i]=0;
    }
    /* l是矩阵的个数，只有两个以上矩阵相乘的时候才会考虑矩阵链乘积的问题，所以直接将l置为2*/
    for (int l = 2; l < A.length;l++)
    {
        /*i从1开始进行，因为本次循环会有l个矩阵参与运算，所以到length - l + 1 的时候循环终止*/
        for (int i = 1;i<A.length-l+1;i++)
        {
            /* 我们把数组m里面每个有效的值置为无限大 j是终结的位置，m[i][j]记录的是从i开始，到j结束的算术乘动作最小值*/
            int j = i + l - 1;
            m[i][j] = MAX;
            /* 我们来遍历所有可能的k，找一下最合适的分割点 */
            for (int k = i ; k< j ;k++)
            {
                int result = m[i][k] + m[k+1][j] + A[i-1]*A[k]*A[j];
                if (result < m[i][j])
                {
                    m[i][j] = result;
                    s[i][j] = k;
                }

            }

        }
    }
    System.out.print("display m :" + "\n");
    display(m);
    System.out.print("---------------" + "\n");
    System.out.print("display s:" + "\n");
    display(s);
    /*因为第一行无效，所以我们实际上返回的是第二行数组中最后一个值 对于样例数据来说，i=1,j=4 */
    return m[1][A.length-1];
}

/**
 * 测试所用，适用于打印m或者s数组
 * @param array
 */
static void display(int [][] array){
    int n = array[0].length;
    for(int i=0;i<n;i++){
        for(int j=0;j<n;j++){

                System.out.print(array[i][j]+" ");
        }
        System.out.print("\n");
    }
}
}
